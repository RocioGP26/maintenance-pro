# MKT-06 · Secuencias de Email y Automatización

**Código:** MKT-06 · Sprint 12.6 · **Entregado**

> El email no existe para vender en cada mensaje. Existe para acompañar al prospecto hasta que descubra el valor de Maintix.

**Toda la operación. Una sola plataforma.**

**Prerequisitos:** [MKT-01 · Identidad y mensajes](01-identidad-mensajes-marca.md) · [MKT-02 · Elevator Pitch](02-elevator-pitch-guiones.md) · [MKT-05 · Landing Page](05-landing-copy.md) · [MCM-06 · Onboarding](/mcm/chapters/06-onboarding-implementacion.md)

**Plataformas de referencia:** Brevo · Mailchimp · HubSpot · ActiveCampaign · etc.

---

## Objetivo del capítulo

Definir las **secuencias oficiales de correo electrónico** para marketing, ventas y éxito del cliente: automatizaciones, mensajes aprobados, frecuencia, objetivos y criterios de conversión.

---

## 1 · Filosofía

Cada correo debe ayudar al prospecto a **avanzar un paso**.

**Nunca** intentar cerrar una venta en el primer email.

La secuencia debe generar:

| | |
|---|---|
| **Confianza** | |
| **Educación** | |
| **Interés** | |
| **Acción** | |

**Objetivo final:** no abrir un correo — sino que el usuario experimente el valor de Maintix y decida convertirlo en la plataforma de su operación.

→ Primera victoria operativa: primera OT · primer producto · dejar de depender solo de Excel.

---

## 2 · Principios

| Principio | Aplicación |
|-----------|------------|
| Hablar del **problema** antes que del producto | Mostrar el dolor operativo |
| **Un objetivo** por correo | Un CTA principal |
| Corto y claro | Lectura en menos de 2 minutos |
| **Personal** | Escribir como una conversación |
| Valor antes que venta | Educar primero |

## 4 · Flujo 1 · Trial de 15 días

**Objetivo:** lograr que el usuario llegue al **primer momento de valor**.

| Día | Asunto / tema | Objetivo | CTA |
|-----|---------------|----------|-----|
| **0** | Bienvenido a Maintix 👋 | Bienvenida · siguiente paso | Ingresar a Maintix |
| **1** | Configura tu empresa | Sede · usuarios · módulo elegido | Completar configuración |
| **3** | Realiza tu primer proceso | OT *(Maintenance)* · producto *(Inventory)* | Según módulo |
| **7** | ¿Cómo va tu prueba? | Dashboard · dudas · ofrecer demo | Agendar una reunión |
| **12** | Tu operación ya comenzó a cambiar | Beneficios · siguiente plan · objeciones | Ver planes |
| **15** | ¿Continuamos? | Cierre trial | Elegir plan · Solicitar extensión |

### Día 0 · Bienvenida

**Asunto:** `Bienvenido a Maintix 👋`

**Contenido:** dar la bienvenida · explicar el siguiente paso · enlace directo al tenant.

**CTA:** Ingresar a Maintix

### Día 1 · Configuración

**Tema:** Configura tu empresa.

Recordar: sede · usuarios · módulo elegido.

**CTA:** Completar configuración

### Día 3 · Primer proceso

**Tema:** Realiza tu primer proceso.

| Módulo | Acción |
|--------|--------|
| **Mantenimiento** | Crea tu primera Orden de Trabajo |
| **Inventario** | Registra tu primer producto |

### Día 7 · Check-in

**Tema:** ¿Cómo va tu prueba?

Contenido: revisar dashboard · resolver dudas · ofrecer demo.

**CTA:** Agendar una reunión

### Día 12 · Valor

**Tema:** Tu operación ya comenzó a cambiar.

Explicar: beneficios obtenidos · siguiente plan · resolver objeciones ([MCM-09](/mcm/chapters/09-manejo-objeciones.md)).

### Día 15 · Conversión

**Tema:** ¿Continuamos?

**CTA:** Elegir plan · o Solicitar extensión

→ [MCM-06 · Onboarding](/mcm/chapters/06-onboarding-implementacion.md) · [MKT-05 · CTAs](05-landing-copy.md#8--conversiones)

## 11 · Personalización

Usar variables cuando existan datos:

```
Hola {{nombre}}

Empresa: {{empresa}}
Sector:  {{sector}}
Módulo:  {{modulo}}
```

**Nunca** enviar correos genéricos cuando existan datos disponibles.

Adaptar día 3 del trial según `{{modulo}}` (Maintenance vs Inventory).

## Filosofía del capítulo

El mejor correo **no** es el que habla más de Maintix.

Es el que ayuda al prospecto a dar el **siguiente paso**.

Cada mensaje debe acercarlo a una primera victoria operativa: crear su primera orden de trabajo, registrar su primer producto o descubrir que ya no necesita depender de Excel para controlar su empresa.
