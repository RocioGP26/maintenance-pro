## Antes

- Mantenimiento reactivo · sin preventivos
- Historial en chats y cuadernos
- Reportes al cliente final tardan días
- Cada reparación deja información que no se conserva

---

## Problema

Cada reparación deja información valiosa. Cuando esa información queda en un cuaderno o en un chat, **el conocimiento desaparece** — y el taller no puede responder al cliente con datos reales.

---

## Implementación

| | |
|---|---|
| **Módulo** | Mantenimiento — activos · OT · evidencia |
| **Onboarding** | Activos de clientes frecuentes · plantilla OT taller |
| **Duración** | 2 semanas hasta historial consultable |
| **Plan** | Start |

---

## Resultado

- Preventivos programados por activo
- Historial completo con fecha · técnico · evidencia
- Respuesta al cliente en la misma llamada
- El historial de cada reparación, siempre disponible

**KPIs orientativos:** Disponibilidad flota · Cumplimiento de visitas · Tiempo de respuesta

---

## Aprendizajes

> «El cliente pregunta cuándo se atendió el equipo 12. Hoy respondemos en la llamada, no en tres días.»

- Fotos y notas en cada cierre de OT
- Cliente valora trazabilidad más que velocidad de software
- Empezar con flota o equipos de mayor facturación
