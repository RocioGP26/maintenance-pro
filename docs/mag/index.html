02 · Autenticación JWT

Una identidad.
Un token.
Una empresa.

JWT identifica usuario, tenant, rol, plan y módulos. Sin reenviar credenciales en cada solicitud.

POST/api/v1/auth/login
{ "token": "<jwt>", "expires_in": 86400, "user": { "id": 15, "nombre": "Ana García", "rol": "admin" }, "empresa": { "id": 4, "slug": "empresa-xyz", "nombre": "Empresa XYZ" } }

HS256 · SECRET_KEY · ±30 s skew · Refresh POST /api/v1/auth/refresh → MAG v2
