# UX · Experiencia integral (Journey)

Cómo se siente usar Maintix **de principio a fin** — no pantalla por pantalla, sino como historia continua.

## Fases del journey

```
Descubrir → Probar → Configurar → Operar → Decidir → Escalar
```

---

## 1 · Descubrir

**Quién:** Dueño, gerente, administrador  
**Canal:** Landing, referido, demo  
**Sensación buscada:** «Esto entiende mi operación»

| Momento | Experiencia |
|---------|-------------|
| Primera visita | Frase «Toda la operación. Una sola plataforma.» + sectores reconocibles |
| Propuesta valor | Casos concretos: OT, stock, ventas — no lista infinita de features |
| CTA | «Empieza gratis» — sin tarjeta, sin llamada obligatoria |

---

## 2 · Probar (Trial)

**Duración emocional:** Primeros 15 minutos definen retención.

```
Registro → Elegir sector → Activar módulos → Primera acción útil
```

| Hito | Meta UX |
|------|---------|
| Registro | &lt; 2 min, email + empresa |
| Onboarding | Wizard MTX-UX-PER-005, no formulario único |
| Primera victoria | Técnico: 1 OT · Vendedor: 1 venta · Gerente: dashboard con dato demo o real |
| Día 1 | Email MTX-EML-013 trial con un solo CTA |

**Sensación:** «Ya estoy operando», no «debo configurar 40 cosas».

---

## 3 · Configurar

**Quién:** Administrador (+ gerente valida)  
**Sensación:** Control sin abrumar

- Usuarios invitados con **rol claro** (no «usuario genérico»)
- Módulos apagados = invisibles en menú
- Campos sectoriales explicados con hint, no manual PDF

---

## 4 · Operar (día a día)

El corazón del producto. Cada perfil vive aquí horas.

| Perfil | Ritmo | Sensación objetivo |
|--------|-------|-------------------|
| Técnico | Interrupciones constantes | Rápido, confiable, sin perder datos |
| Bodeguero | Alto volumen transaccional | Precisión, trazabilidad |
| Vendedor | Cliente delante | Fluidez, cero espera |
| Gerente | Miradas de 30 s–5 min | Claridad, alertas accionables |

**Principio transversal:** La complejidad vive en el sistema (valores Maintix), no en la pantalla.

---

## 5 · Decidir

**Quién:** Gerente, dueño  
**Herramientas:** Dashboard, reportes, MRL PDF

Journey de reporte:

```
Alerta en dashboard → Detalle → Exportar PDF → Compartir en reunión
```

Sin re-exportar a Excel como paso obligatorio.

---

## 6 · Escalar

**Señales:** Más sedes, más usuarios, más módulos  
**UX:** Mismo producto, más scope — no «versión enterprise» distinta

- Multi-sede: selector visible en navbar
- IAM: roles que mapean a los 5 perfiles
- Facturación: transparente en configuración

---

## Mapa emocional

| Fase | Emoción deseada | Emoción a evitar |
|------|-----------------|------------------|
| Descubrir | Curiosidad confiada | Sospecha de hype |
| Probar | Logro rápido | Abrumamiento |
| Configurar | Control | Parálisis |
| Operar | Flujo | Fricción, miedo a errores |
| Decidir | Seguridad en datos | Duda sobre veracidad |
| Escalar | Crecimiento natural | Migración forzada |
