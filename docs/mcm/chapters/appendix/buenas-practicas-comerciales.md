## 2 · Embudo comercial (resumen)

```
Awareness → Interés → Calificación → Demo → Trial → Cierre → Expansión
```

| Etapa | Herramienta MCM | KPI orientativo |
|-------|-----------------|-----------------|
| Awareness | Landing · contenido · MCM-02 | Visitas · registros trial |
| Calificación | ICP · MCM-03 sectores | % banda A/B |
| Demo | PLAY-001–005 | Demo → trial |
| Trial | MCM-06 onboarding | Activación día 7 |
| Cierre | MCM-04 planes · OBJ | Trial → pago |
| Expansión | Segundo módulo · upgrade | NRR · OMI ↑ |

## 3 · Mensajes que sí / mensajes que no

| ✅ Decir | ❌ Evitar |
|---------|----------|
| **Slogan:** «No necesitas otro software…» | Features sin contexto |
| **Una frase** + elevator pitch (MCM-01) | Improvisar definición en reunión |
| Transformación (MCM-02) | Guerra de features vs competidor |
| Trial 15 días · Start | «ERP completo desde el día 1» |
| Mantenimiento **e** Inventario en producción | Vender CRM/Finance como disponibles |
| MRG documenta el producto | Prometer fechas de roadmap |

## 4 · Demo de clase mundial

1. **PLAY-001** — preparar sector, DMU, historia MCM-03  
2. **PLAY-002** — apertura con dolor, no con login  
3. **PLAY-003** — un flujo completo en 10–15 min  
4. **PLAY-004** — KPI del sector  
5. **PLAY-005** — plan + trial + siguiente paso  

**Regla:** si el prospecto no ve su operación reflejada, la demo falló — aunque el producto funcionó perfecto.

## 5 · Post-venta y expansión

| Señal | Acción |
|-------|--------|
| Piden segundo módulo | Proponer **Grow** |
| Nueva sede | Proponer **Scale** |
| >80% usuarios activos | Revisión de valor · caso MTX-CASE |
| Champion cambia de rol | Re-identificar DMU |
| Uso bajo día 14 | Intervención onboarding (MCM-06) |

**Pilar de marca:** *La transformación comienza con un módulo. El crecimiento ocurre dentro de una sola plataforma.*
