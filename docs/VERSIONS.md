## Semver por manual

| Bump | Cuándo | Ejemplo |
|------|--------|---------|
| **MAJOR** `v2.0` | Reestructura, cambio de posicionamiento | MBB v2.0 (separación MDL) |
| **MINOR** `v1.1` | Capítulo nuevo, sección amplia | MPA v1.0 → v1.1 (nuevo capítulo) |
| **PATCH** `v1.0.1` | Typo, enlace, aclaración menor | MPA v1.0.1 (refino MPA-01) |

## Al actualizar una versión

1. Editar **este archivo** (`VERSIONS.md`)
2. Entrada en `{manual}/changelog.md`
3. Si afecta la suite → [changelog.md](changelog.md)
4. Bump en README del manual + catálogo HTML si existe
5. Release interno → tag `docs-vX.Y`

→ Política completa: [VERSIONING.md](VERSIONING.md)
